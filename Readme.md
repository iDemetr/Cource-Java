# Решение курса по изучению Java

Данный репозиторий содержит решения лабораторных работ по курсу Java. Каждая лабораторная работа представляет собой самостоятельный небольшой проект, демонстрирующий различные аспекты программирования на Java

## Используемые технологии
- Java 24, но допустимы 17+
- Swing, JFree.Chart
- Gradle для сборки и управления зависимостями
- Многопоточность, клиент-серверная архитектура
- Парадигмы: Императивное, объектно-ориентированное, частично функциональное (лямбда-выражения)
  
## Структура проекта
В папке ```lib/src/main/java``` располагается множество самостоятельных модулей в папках ```ЛР_**N***```, где ```N``` - номер задания.
Большинство модулей реализует паттерн MVC.

* ```Л1``` - Сравнение времени и точности вычисления квадратного коря методом Ньютона и стандартной функции из Math.
* ```Л2``` - Реализация сложной стуктуры классов(```Forest```, ```Plants```, ```Animals```) для раскрытия парадигмы ООП
* ```Л3``` - Релизует базу данных и консольный интерфейс для работы с ней.
  * CRUD операции (Create, Read, Update, Delete)
  * properties для кеширования данных пользователя для авторизации (классы ```Settings```. ```Authentication``` и ```AuthenticationController```)
  * Поддерживает режим логирования операций
* ```Л4``` - Выполняет анализ затрачиваемого времени на работу ```ArrayList``` и ```HashMap``` во время добавления и удаления элементов.
  * Реализация генерации случаного леса, для проведения анализа, на основе паттерна ```Factory```.
  * Реализация доступа к защищённым данным о внутреннем буфере ```ArrayList``` (класс ```ArrayListUtils```)
  * Продвинутая система логгирования, для формирования выходных таблиц с информацией о времени, операции и коллекции.
* ```Л5``` - Построение графиков, для анализа сформированных таблиц в результате работы программы Л4.
  * На основе двух ```DataSet``` о среднем и общем времени выполнения операции формируется по графику для ArrayList и HashMap
  * Примитивно реализованный менеджер тем (```ThemeManager```), для смены стиля приложения.
* ```Л6``` - Тренажёр слепой печати.
  * Подробнее в ```(ссылка в будующем)```
  * Swing
  * Реализация паттерна ```Observer```
  * Поддержка множества режимов и сложностей
  * Реализация самостоятельных UI-компонентов
  * Киллер-фича - интерактивный "живой" интерфейс, где текст динамически течёт через центр экрана, обеспечаивая:
    * Ествественный фокус на текущем символе
    * Визуальную обратную связь
    * Эстетичный эффект постоянного движения, без сильных скачков.
* ```Л7``` - Работа с многопоточностью, решение проблемы гонки данных.\
  **Задача** - Запустив N и M потоков на чтение и запись обеспечить синхроизированный, последовательный (**шахматный**) доступ на выполнение операций.\
  Найдено 3 рабочих решения:
    * ```SafetyString``` - ждать, если идёт запись или доступных на чтение данных нет, не писать, пока все читатели не дочитают (```wait```, ```sleep```, ```interrupt```)
    * ```SafetyString2``` - на связке ```ReentrantReadWriteLock```, ```readLock``` и ```writeLock```
    * ```SharedResource``` - самое лакониченое, на связке ```ReentrantLock``` и ```Condition```
  
  Решения сугубо индивидуальные, подогнанные под строгие условия задачи.
* ```Л8``` - Релазиация клиент-сервера на примере примитивного чата
  * Поддерживается история чата
  * Отслеживание "онлайна" собеседников
  * Небольшая модерация со стороны сервера

## Как запустить

1. Клонируйте репозиторий:  
   ```bash
   git clone https://github.com/iDemetr/Cource-Java.git
   cd Cource-Java
   ```
2. Перейдите в директорию нужной лабораторной работы, например:
    ```bash
    cd lab1
    ```
3. Запустите проект с помощью Gradle Wrapper (рекомендуется):
    ```bash
    ./gradlew run
    ```
	Для Windows:
	```bash
  	gradlew.bat run
  	```

5. Либо откройте проект в вашей IDE, например ```Eclipse```, и запустите любой модуль с методом main.
